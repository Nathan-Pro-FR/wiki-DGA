Pipeline

Fonctionnement de A à Z

Chaque média posté dans le salon Discord parcourt ce pipeline automatiquement. Clique sur chaque étape pour voir les détails techniques. Voici le chemin que prend ta photo entre le moment où tu l'envoies sur Discord et le moment où elle apparaît sur le site. Clique sur chaque étape pour en savoir plus.

01
Tu postes un média DISCORD
Tu envoies une image, gif ou vidéo dans le salon désigné. C'est la seule action humaine requise. Tu envoies une photo ou une vidéo dans le bon salon Discord, comme tu le ferais normalement. C'est tout ce que tu as à faire.
Formats supportés : .jpg .png .gif .webp .mp4 .mov .webm .mkv et plus. La détection se fait par MIME type ou extension d'URL. Tous les formats courants marchent : photos, GIFs animés, vidéos… Le système reconnaît automatiquement le type de fichier, tu n'as rien à préciser.
02
GitHub Actions se déclenche AUTOMATION
Toutes les heures, un workflow s'exécute automatiquement sur les serveurs GitHub. Il peut aussi être lancé manuellement. Toutes les heures, un petit robot hébergé par GitHub se réveille tout seul et commence à travailler. On peut aussi le réveiller à la demande.
Déclencheur : cron: "0 * * * *" (chaque heure pile) + workflow_dispatch pour un lancement manuel depuis l'onglet Actions de GitHub. C'est comme un réveil programmé toutes les heures. Si on ne veut pas attendre, il y a aussi un bouton "lancer maintenant" sur le site GitHub du projet.
03
Scan de l'historique Discord API
Le bot lit jusqu'à 1 500 messages en remontant l'historique par paquets de 100 (pagination). Le robot relit jusqu'aux 1 500 derniers messages du salon Discord et récupère toutes les images et vidéos qu'il trouve.
Le script recupere_medias.js interroge l'API Discord v10 avec le token bot. La pagination évite la limite de 100 messages par requête. Les doublons sont filtrés par attachment.id. Discord ne donne pas plus de 100 messages à la fois, alors le robot en demande plusieurs "pages" les unes après les autres — comme feuilleter un livre. Il évite aussi de récupérer deux fois la même image.
04
Comparaison intelligente DIFF
Le workflow compare l'ancienne liste avec la nouvelle pour détecter ajouts, suppressions et jetons rafraîchis. Le robot compare la liste de photos actuelle avec celle qu'il avait la dernière fois, pour savoir ce qui a changé : nouvelles photos, photos supprimées, ou liens à mettre à jour.
Les médias sont identifiés par un ID stable extrait de l'URL (pas l'URL complète qui change). Cela permet de détecter les rafraîchissements de jetons Discord CDN sans les confondre avec de nouveaux médias. Chaque image a un numéro unique permanent. Discord change parfois l'adresse exacte d'une image pour des raisons de sécurité, mais le numéro reste le même — le robot fait donc la différence entre "nouvelle image" et "même image, lien mis à jour".
05
Mise à jour du fichier de données GIT
Le fichier donnees.json est mis à jour sur GitHub. Si rien n'a changé, aucun commit n'est créé. Le robot met à jour la liste des photos sur GitHub. S'il n'y a rien de nouveau, il ne fait rien et ne laisse pas de trace inutile.
Le workflow vérifie avec git diff --quiet avant de committer. Cela évite de polluer l'historique Git avec des commits vides à chaque run horaire. Imagine un cahier où on note les changements. Si rien n'a bougé, on n'écrit rien — le cahier reste propre et lisible. Ici c'est pareil avec l'historique GitHub.
06
Redéploiement automatique du site GITHUB PAGES
Un nouveau commit déclenche GitHub Pages qui redéploie le site en quelques secondes. Dès qu'une photo est ajoutée à la liste, le site web se reconstruit automatiquement en quelques secondes pour l'afficher.
Le frontend (index.html + script.js) lit donnees.json au chargement de la page et affiche dynamiquement tous les médias dans la grille cyberpunk. Quand tu ouvres le site, il lit automatiquement la liste des photos à jour et les affiche dans la grille. Pas besoin de recharger manuellement — ça se fait tout seul.
07
Notification Discord envoyée WEBHOOK
Un embed est posté dans le salon avec les stats de la synchronisation et un lien direct vers la galerie. Une fois que tout est fait, le robot envoie un message récapitulatif dans le salon Discord : combien de photos ajoutées, combien supprimées, et un lien vers le site.
L'embed contient : ajouts ➕, suppressions ➖, jetons rafraîchis 🔄, total 📊, lien galerie et lien rapport GitHub Actions. La date utilise les timestamps Discord natifs (<t:unix:F>) pour l'heure locale de chaque membre. Le message de résumé affiche le bilan de la mise à jour et un bouton pour aller voir la galerie directement. L'heure affichée s'adapte automatiquement au fuseau horaire de chaque membre du serveur.

Terminologie

Les concepts clés

Ce projet combine plusieurs technologies modernes. Voici leur rôle. Voici les outils utilisés dans ce projet, expliqués simplement.

⚡
CI/CD
Continuous Integration / Deployment. Le principe général derrière l'automatisation du pipeline complet. C'est l'idée de tout automatiser : quand quelque chose change, tout se met à jour tout seul, sans que tu aies à intervenir.
🤖
GitHub Actions
L'outil qui exécute le workflow automatiquement sur des serveurs GitHub. Gratuit pour les repos publics. Un robot gratuit hébergé par GitHub qui peut exécuter des tâches automatiquement selon un planning ou un déclencheur.
🌐
GitHub Pages
Hébergement web statique gratuit directement depuis un repo GitHub. Chaque push redéploie le site. Un hébergement de site web gratuit fourni par GitHub. Dès qu'on modifie les fichiers du projet, le site se met à jour automatiquement.
🔀
GitOps
Git est la source de vérité centrale. Tout changement passe par un commit, ce qui déclenche le reste. Tout passe par GitHub : c'est la "mémoire centrale" du projet. Chaque modification enregistrée là-bas déclenche automatiquement la mise à jour du site.
📡
API Polling
Le bot interroge régulièrement l'API Discord pour récupérer les nouveaux messages à intervalle fixe. Le robot "demande" à Discord régulièrement : "est-ce qu'il y a du nouveau ?" — comme regarder ta boîte aux lettres toutes les heures.
🔗
Webhook Discord
Une URL spéciale qui permet d'envoyer des messages dans un salon Discord depuis une application externe. Une adresse secrète qui permet à notre robot d'envoyer des messages dans le salon Discord, comme s'il était un membre du serveur.

Sécurité

Conçu pour rester privé

Le salon source Discord reste privé. Seules les URLs CDN publiques des médias sont exposées. Ton salon Discord reste privé. Le site ne montre que les images — pas les messages, pas les membres, rien d'autre.

🔒
BOT EN LECTURE SEULE

Le bot Discord ne peut que lire les messages. Il n'a aucun droit d'écriture, de suppression ou de gestion du serveur. Le robot peut seulement regarder les messages — comme quelqu'un qui peut lire mais pas écrire. Il ne peut ni supprimer, ni modifier quoi que ce soit sur ton serveur.

🗝️
GITHUB SECRETS

Le token bot, l'ID du salon et l'URL du webhook sont stockés dans les Secrets chiffrés de GitHub — jamais dans le code source. Les mots de passe et clés d'accès sont stockés dans un coffre-fort sécurisé sur GitHub — jamais écrits dans les fichiers du projet que tout le monde peut voir.

🛡️
SALON SOURCE PRIVÉ

Le salon Discord reste privé. Seules les URLs publiques du CDN Discord sont référencées dans le fichier de données. Ton salon Discord reste entre membres. Le site affiche seulement les images, sans révéler qui les a postées ni le contenu des messages autour.

🔄
JETONS CDN RAFRAÎCHIS

Discord fait expirer les URLs CDN pour des raisons de sécurité. Le pipeline détecte et met à jour ces jetons automatiquement. Discord change régulièrement les liens des images pour des raisons de sécurité. Notre système détecte ces changements et met les liens à jour automatiquement — les images ne disparaissent jamais du site.
