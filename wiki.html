Pipeline

Fonctionnement de A à Z

Chaque média posté dans le salon Discord parcourt ce pipeline automatiquement. Clique sur chaque étape pour voir les détails techniques.

01
Tu postes un média DISCORD
Tu envoies une image, gif ou vidéo dans le salon désigné. C'est la seule action humaine requise.
Formats supportés : .jpg .png .gif .webp .mp4 .mov .webm .mkv et plus. La détection se fait par MIME type ou extension d'URL.
02
GitHub Actions se déclenche AUTOMATION
Toutes les heures, un workflow s'exécute automatiquement sur les serveurs GitHub. Il peut aussi être lancé manuellement.
Déclencheur : cron: "0 * * * *" (chaque heure pile) + workflow_dispatch pour un lancement manuel depuis l'onglet Actions de GitHub.
03
Scan de l'historique Discord API
Le bot lit jusqu'à 1 500 messages en remontant l'historique par paquets de 100 (pagination).
Le script recupere_medias.js interroge l'API Discord v10 avec le token bot. La pagination évite la limite de 100 messages par requête. Les doublons sont filtrés par attachment.id.
04
Comparaison intelligente DIFF
Le workflow compare l'ancienne liste avec la nouvelle pour détecter ajouts, suppressions et jetons rafraîchis.
Les médias sont identifiés par un ID stable extrait de l'URL (pas l'URL complète qui change). Cela permet de détecter les rafraîchissements de jetons Discord CDN sans les confondre avec de nouveaux médias.
05
Mise à jour du fichier de données GIT
Le fichier donnees.json est mis à jour sur GitHub. Si rien n'a changé, aucun commit n'est créé.
Le workflow vérifie avec git diff --quiet avant de committer. Cela évite de polluer l'historique Git avec des commits vides à chaque run horaire.
06
Redéploiement automatique du site GITHUB PAGES
Un nouveau commit déclenche GitHub Pages qui redéploie le site en quelques secondes.
Le frontend (index.html + script.js) lit donnees.json au chargement de la page et affiche dynamiquement tous les médias dans la grille cyberpunk.
07
Notification Discord envoyée WEBHOOK
Un embed est posté dans le salon avec les stats de la synchronisation et un lien direct vers la galerie.
L'embed contient : ajouts ➕, suppressions ➖, jetons rafraîchis 🔄, total 📊, lien galerie et lien rapport GitHub Actions. La date utilise les timestamps Discord natifs (<t:unix:F>) pour l'heure locale de chaque membre.

Terminologie

Les concepts clés

Ce projet combine plusieurs technologies modernes. Voici leur rôle.

⚡
CI/CD
Continuous Integration / Deployment. Le principe général derrière l'automatisation du pipeline complet.
🤖
GitHub Actions
L'outil qui exécute le workflow automatiquement sur des serveurs GitHub. Gratuit pour les repos publics.
🌐
GitHub Pages
Hébergement web statique gratuit directement depuis un repo GitHub. Chaque push redéploie le site.
🔀
GitOps
Git est la source de vérité centrale. Tout changement passe par un commit, ce qui déclenche le reste.
📡
API Polling
Le bot interroge régulièrement l'API Discord pour récupérer les nouveaux messages à intervalle fixe.
🔗
Webhook Discord
Une URL spéciale qui permet d'envoyer des messages dans un salon Discord depuis une application externe.

Sécurité

Conçu pour rester privé

Le salon source Discord reste privé. Seules les URLs CDN publiques des médias sont exposées.

🔒
BOT EN LECTURE SEULE

Le bot Discord ne peut que lire les messages. Il n'a aucun droit d'écriture, de suppression ou de gestion du serveur.

🗝️
GITHUB SECRETS

Le token bot, l'ID du salon et l'URL du webhook sont stockés dans les Secrets chiffrés de GitHub — jamais dans le code source.

🛡️
SALON SOURCE PRIVÉ

Le salon Discord reste privé. Seules les URLs publiques du CDN Discord sont référencées dans le fichier de données.

🔄
JETONS CDN RAFRAÎCHIS

Discord fait expirer les URLs CDN pour des raisons de sécurité. Le pipeline détecte et met à jour ces jetons automatiquement.
